Usage Rules

  • Navy #18243a — app header/nav background only
  • Blue #3560c8 — primary action buttons, selected states, active tab indicator, left-border on selected rows, key accents
  • Sky #6899e2 — secondary labels (source type, subtle meta info), step indicator when active, dot plot secondary dots
  • Green #1e7d58 — success states only: HEALTHY badge, high stability values (≥ 0.85), consensus checkmarks, Download button, completed pipeline steps
  • Pale BG #eaf2fc — card headers, selected row backgrounds, "why kept" inset box, active tab background
  • No orange, no red, no purple in the main UI. Red is reserved for downregulated direction arrow only.

Interaction Notes

Click dataset chip Highlights selected chip (blue border + pale-blue bg). Updates all downstream result references to selected dataset name.
Click pipeline card Selects pipeline A or B. If Pipeline B selected, Pipeline B tab in Advanced Settings becomes relevant.
Click Run / Load Results Transitions to Screen 2 (Progress). If results already cached for this configuration, skips progress and goes directly to Screen 3.
Click table row (Screen 3) Highlights row (blue left border + pale-blue bg). Updates: selected pathway detail panel, dot plot highlighted dot, Why kept text, gene list.
Click chart tab Switches between Dot plot, Pipeline funnel, Redundancy network. Each tab shows a distinct visualization of the shortlist.
Click Advanced settings row Opens Screen 4 modal over the current screen (Setup or Results). Backdrop dims the underlying screen.
Click Advanced Settings tab Switches active parameter group. Only the active tab's fields are shown in the content area.
Click Apply & Run Saves custom parameters, switches preset to "Custom", closes modal, triggers new analysis run (→ Screen 2).
Click Download Results Triggers download of results CSV/Excel. Available only on Screen 3 when results are loaded.

R/Shiny Implementation Notes

App Structure

Use bslib or a custom CSS theme — do not use default Shiny Bootstrap styling. Apply custom CSS to match the design tokens above.
The 4 screens map to a single-page app with uiOutput/renderUI driving screen transitions, or use shinyjs to show/hide screen divs.
Nav bar: fixed top, tags$nav with inline CSS. Height 50px. No default Shiny navbar.

Screen 1 — Setup

Detect datasets: list.dirs("data/", full.names=FALSE, recursive=FALSE) on app start. Populate dataset chips reactively.
Preset selector: radioButtons styled as pill chips via CSS, or custom JS-driven chip selector.
Advanced settings: shinyjs::hidden() div, toggled with shinyjs::toggle() on click.
Run button: actionButton — check for cached RDS result before triggering analysis.

Screen 2 — Progress

Use withProgress + incProgress or a custom reactive polling pattern with reactiveTimer.
Pipeline log: render a renderUI list of step divs, updating state from a reactive values list as each stage completes.
Cancel: observeEvent on cancel button, interrupt via future/promises if using async, or flag a reactive that the main loop checks.

Screen 3 — Results

Summary cards: valueBox (bslib) or custom renderUI with inline CSS matching the 3px top border design.
Shortlist table: DT::datatable with selection='single', custom CSS to match row height (48px) and selected-row styling. Set dom='t' to suppress default DT controls.
Row click → detail panel: observeEvent(input$table_rows_selected, ...) → renderUI detail panel and re-render dot plot with highlighted term.
Dot plot: ggplot2 with renderPlot or plotly::renderPlotly. Map −log₁₀(q) to x-axis, term to y-axis (reordered by rank), gene count to dot size, stability to colour.
Chart tabs: tabsetPanel styled with CSS to match the tab bar design. Three panels: Dot plot, Pipeline funnel, Redundancy network.
Download: downloadButton + downloadHandler, exporting results_df as .csv and optionally .xlsx via openxlsx.

Screen 4 — Advanced Settings

Modal: showModal(modalDialog(...)) or a custom absolutePanel overlay with shinyjs show/hide.
Tab nav: custom HTML tabs with JS click handlers, or tabsetPanel with heavy CSS override.
Sliders: sliderInput with custom CSS (override Shiny's default slider track/thumb to match design).
Toggles (Pipeline B): checkboxInput styled as a CSS toggle via label::before / label::after trick, or use shinyWidgets::switchInput.
Apply & Run: save params to reactiveValues, set rv$preset <- "Custom", close modal, trigger analysis.
Reset: restore all inputs to hardcoded defaults using updateSliderInput, updateNumericInput, etc.

Colour/CSS Integration

Add a www/styles.css file with CSS custom properties matching all design tokens above. Reference in ui.R via includeCSS("www/styles.css").
Import DM Mono from Google Fonts: tags$link(rel="stylesheet", href="https://fonts.googleapis.com/css2?family=DM+Mono:wght@400;500").
Disable Bootstrap's default button/form styling for custom-styled inputs. Use bslib::bs_theme() with version=5 as baseline.
